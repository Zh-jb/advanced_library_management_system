# 评分细则对应说明

## 1. 系统功能实现

本系统围绕“图书管理系统”核心业务完整实现：

- 图书增删查改：`/api/books` 支持新增、删除、分页查询、修改。
- 借还书记录：`/api/borrow-records` 支持借书登记、查询记录，`/api/borrow-records/{id}/return` 支持归还。
- 读者信息管理：`/api/readers` 支持读者新增、修改、删除、查询、冻结账号。
- 逾期提醒：`/api/overdue` 查询逾期，`/api/reminders/generate` 生成提醒。
- 管理员权限：管理员可管理全部数据，普通读者只能查看和操作自己的借阅记录。

## 2. 编码规范

- 采用前后端分离思路，前端位于 `frontend/`，后端位于 `backend/app/`。
- 后端分层清晰：
  - `main.py`：API 路由。
  - `schemas.py`：输入校验。
  - `services.py`：业务逻辑。
  - `database.py`：数据库建表、索引、视图。
  - `security.py`：密码哈希与登录认证。
- SQL 使用参数化查询，避免 SQL 注入。
- 函数命名与变量命名语义明确。

## 3. 用户界面与交互

- 提供登录页、侧边栏导航、图书管理、读者管理、借还记录、逾期提醒、数据总览等页面。
- 前端表单包含必填校验、数值校验、错误提示。
- 操作成功或失败均使用右上角 Toast 提示。
- 管理员与普通读者页面权限自动区分，避免普通读者误操作。

## 4. 数据操作与查询效率

数据库采用 SQLite，并进行了以下优化：

- 图书表建立 `idx_books_title`、`idx_books_category`、`idx_books_isbn` 索引。
- 借阅记录表建立 `idx_records_reader_status`、`idx_records_due_date`、`idx_records_book` 索引。
- 使用 `v_borrow_detail` 视图整合图书、读者、借阅记录，减少重复 JOIN 逻辑。
- 列表接口采用分页参数 `page`、`page_size`，避免一次性返回过多数据。
- 删除图书、删除读者前会检查是否存在未归还记录，保证数据一致性。

## 5. 创新与拓展

本项目在基础功能外加入了可作为亮点展示的拓展功能：

1. **大数据可视化看板**：
   - 馆藏分类分布图。
   - 近 14 天借阅趋势图。
   - 热门图书 Top10。
   - 馆藏数量、读者数量、借阅中、逾期、已归还指标卡片。

2. **CSV 数据导出**：
   - 图书数据导出。
   - 借还记录导出。

3. **权限分级控制**：
   - 管理员拥有完整管理权限。
   - 普通读者只能查看自己的借阅记录，不能管理图书和读者。

4. **逾期提醒中心**：
   - 自动识别逾期记录。
   - 一键生成提醒消息。
   - 归还后自动将提醒标记为已处理。

5. **可扩展架构**：
   - 当前使用 SQLite 便于课程提交。
   - 后续可迁移 MySQL / PostgreSQL。
   - 可继续扩展邮件提醒、短信提醒、智能推荐、扫码借阅等功能。
